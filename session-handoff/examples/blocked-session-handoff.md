# Example: blocked session handoff

```md
# Handoff

## Current goal
- Create an Opencode skill for session handoff.

## Sub-goals
- Review the proposed handoff outline
- Confirm target audience and scope
- Produce the final skill package

## Current status
- Best-practice research and draft review are complete.
- Final skill creation is waiting on one user decision.

## Established context
- The user wants a compact handoff document, not a transcript.
- They care about preserving current goal, context, artifacts, decisions, blockers, and next steps.

## Artifacts
- No files created yet.

## Decisions made
- Decision: recommend adding current status, open questions, risks, and one recommended next step
  - Why: these improve restart speed and reduce ambiguity for the next person

## Open questions / assumptions
- Open question: should the skill be general-purpose or coding-session specific?

## Blockers
- Waiting on user input: confirm desired scope of the skill.

## Risks / caveats
- If scope is guessed incorrectly, the resulting skill may be either too broad or too narrow.

## Recommended next step
- Ask the user to choose between general-purpose and coding-focused scope.
```
